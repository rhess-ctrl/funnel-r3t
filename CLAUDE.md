## Commands

```bash
yarn dev          # Start dev server with concurrent Tailwind watch + Vite (localhost:5173)
yarn build        # Generate tw-version, format, then Vite production build
yarn format       # Prettier format (CSS, PCSS, JS, JSX)
yarn cc           # Check formatting with Prettier (no write)
yarn spell        # CSpell spell check
yarn deploy       # Increment patch version, build, and deploy to Vercel
```

No test command is configured in package.json; `App.test.js` exists but is outdated.

## Architecture

Single-page React app that renders an interactive animated SVG funnel chart with color palette switching and smooth tweening.

**State flow:**
- `App.jsx` (class component) — owns all state (selected color, bar values, animation state)
- `AppContext.jsx` — provides state + callbacks to the tree via Context API
- Components in `src/components/` consume context with `useContext`

**Animation system:**
- `Bar.jsx` runs two concurrent `between.js` tweens per bar: color transition (800ms) and height change (1000ms cubic easing)
- `src/utils/ColorPlugin.js` — custom Between.js plugin for interpolating CSS color strings via the `color` + `color-string` libraries

**Key utilities:**
- `src/utils/funnelDefs.js` — geometry config for the 5 funnel sections (SVG coordinates)
- `src/utils/namedColors.js` — 20 color palettes (clrs.cc + custom)
- `src/utils/tw-version.js` — auto-generated by the `.generate` script; do not edit manually

**Styling:**
- Tailwind CSS 4 with PostCSS; input is `src/styles/input.pcss`, compiled output is `src/styles/app.css`
- Custom color plugin `proto-tailwindcss-clrs` maps named palettes to Tailwind utilities
- Tailwind preflight is disabled in `tailwind.config.js`

**Deployment:** Vercel — `yarn deploy` runs `predeploy` (auto-increments patch version via `npm version patch`, then builds) before pushing.
